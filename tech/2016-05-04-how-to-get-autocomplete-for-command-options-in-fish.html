Fish has support to autocomplete the options of any command at the shell. This is sometimes called tab completion.

For example, when I type ls - and TAB, Fish shows me the options available for the ls command. This works even when youâ€™ve typed out part of an option. Fish generates this information by parsing all the man pages installed on your system. Fish also ships with completions for most of the common commandline tools used in Linux.

  • The completions that ship with Fish can be seen as files in the /usr/share/fish/completions directory. On my system, I found more than 700 such files.

  • The completions that are generated from man pages after Fish is installed or used can be seen as files in the $HOME/.local/share/fish/generated_completions directory. In my home directory, I found about 1500 such files.

  • If you find that Fish autocomplete is not working for the options of a command, that can be fixed easily by asking Fish to reindex from man pages. The Fish function that you need to run for this is: fish_update_completions.

I recently installed Git. Since Git is infamous for its myriad command-line options I missed having autocomplete at the shell for Git command options. After running fish_update_completions, I could get autocomplete for options of all Git commands. The icing on top was to discover that Fish could autocomplete options even for my Git aliases.

Tried with: Fish 2.2.0
