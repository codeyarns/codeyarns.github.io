A while ago Ubuntu added support for Raspberry Pi and so recently I decided it was time to try something other than Raspbian on my Raspberry Pi 3 Model B+ board. Since this was an older (and thus less powerful) RPi board, I chose to install Ubuntu Server instead of Ubuntu Desktop which needs more resources.

Note that Ubuntu has a guide that lists the steps at a high level to install Ubuntu Server on RPi, which turned out to be not detailed enough for me to get my install working. The steps that eventually worked for me are listed below:

  • I downloaded and installed the Raspberry Pi Imager for Windows from here.
  • I plugged in a microSD card into my Windows computer. The Ubuntu guide lists 4GB as sufficient and 8GB as recommended. I initially used an 4GB microSD card that was quite slow, so I switched to a fast 64GB microSD card for my final install. Since my Windows computer does not have a microSD card reader slot, I used a Anker USB-C 2-in-1 SD card reader which worked out ok.
  • In the Imager, I chose Ubuntu Server 20.04.1 LTS 64-bit to install on the microSD card. The Imager pulls the image from the web to install the first time, so the writing step can be slow. The OS image is cached locally, so if you do the same install again the writing step will be fast. After writing, the Imager also does a verifying step to ensure the install was done correctly.
  • I ejected the microSD card from my Windows computer and reinserted it. I created an empty file named ssh on the card. This enables SSH access to the RPi board on its first boot.
  • I ejected the microSD card from my Windows computer and inserted it into my RPi board. I connected the RPi board to my home wireless router using a LAN cable. I finally connected the RPi board to power using a Adafruit 5V 2.5A microUSB cable.
  • I left the RPi to boot Ubuntu for the first time and finish its initialization. After 10 minutes (after the blinking green light had settled into a slow steady blink), I unplugged and plugged back the power to reboot the RPi.
  • I opened up my wireless routerâ€™s web interface and noticed that my RPi board had been assigned an IP address. I had a refresh my routerâ€™s interface a few times and waited 5-10 minutes for this change.
  • I opened WSL on my Windows computer and tried my first SSH login to the RPi board ssh ubuntu@<IP address>. The initial password is ubuntu. Ubuntu immediately asks you to change your password for the ubuntu user.
  • After this I was able to SSH to the RPi normally. I did a sudo apt update ; sudo apt upgrade to update the packages.
