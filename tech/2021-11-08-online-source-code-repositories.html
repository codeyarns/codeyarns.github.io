Sometimes, I like looking up the definition of classes or functions from libraries or tools I use to understand how they are actually implemented. Back in the day, this meant figuring out how to get the source package of that library, downloading it and then searching inside its source files. But now that almost all libraries share their code on Github (or similar services), it has become quite convenient to look up source code repositories. The recent addition of the ability to open a Github repository using VS Code running in the browser is the icing on the cake!

Here are the source code repositories that I most frequently consult:

  • GCC: Mirror of GCC. I use this to consult the C headers and C++ STL that ships with Linux.
  • GLibC: Mirror of the GNU C Library source code.
  • Microsoft STL: Microsoft‚Äôs implementation of the C++ STL.
  • Linux kernel: Linus Torvalds‚Äô fork of the Linux kernel source code.
  • CPython: The implementation of the standard distribution of Python is implemented in C.
  • Vim: Source code of Vim.
  • CUDA Samples: Easiest way to check usage of CUDA API is to check how it is used in CUDA samples.
  • Triton Inference Server
