Slurm (Simple Linux Utility for Resource Management) is a Linux job scheduler commonly used on compute clusters and supercomputers.

Common options

  • -A or --account: Use the specified account for the Slurm command. For example --account=biology_team.

sinfo

This command prints information about the nodes in the cluster.

  • List all nodes: sinfo.

  • Print total number of nodes: sinfo -o "%D".

  • Print all possible states of nodes in the cluster: sinfo -h -o "%T" | sort -u.

  • List the nodes in a specified state. For example, to list idle nodes: sinfo -t idle.

sbatch

This is the command to submit a script to Slurm to run on the next node that becomes available. This command exits immediately printing out the Slurm job ID.

Common options passed to this command:

  • -o or --output: File to write standard output to. For example -o out-%j.txt writes to a filename with the current Slurm job ID.

  • -e or --error: File to write standard error to. For example -e err-%j.txt writes to a filename with the current Slurm job ID.

  • -G or --gpus: Specify the number of GPUs required. For example --gpus=4.

  • --wrap: Specify the command to wrap into a dummy script. Since Slurm looks for a script file by default, --wrap is a trick to use when you just want to run a command on the node. For example --wrap="hostname" prints the hostname in the node we get and exits.

squeue

This command prints out the entire list of Slurm jobs in the queue right now.

  • -u or --user: List the jobs of the specified user. For example --user=joe.

scct

This command can be used to find information about jobs that are already completed.

  • -j or --jobs: Find information of the specified jobs. For example -j 123456.
