Feh Image Viewer

Ashwin Nanjappa

2015-03-10 15:17:09+00:00

Feh is probably the fastest and the most minimal image viewer on Linux. You might want to try it if you need to open and browse through images quickly on even the slowest computers.

Install

Feh can be installed easily:

$ sudo apt install feh

Usage

To open a single image:

$ feh foo.jpg

To open a set of images:

$ feh *.jpg

Keyboard shortcuts

The keyboard shortcuts of Feh that I typically use:

Left: Next image
Right: Previous image
Up: Zoom in
Down: Zoom out
/: Fit window
Esc: Close

You can set your own keyboard shortcuts by creating the file ~/.config/feh/keys. An example configuration file can be seen here. The config options can be seen in man feh.

Options

The command line options that I usually use:

  • --geometry 1920x1080: Start in a window of the specified size. Note that there is no option to open in a maximized window.

  • -F: Start in fullscreen mode

  • -Z: Zoom images to fit window

  • --scale-down: Scale down image to fit window, only when it is bigger than window

Open with

  • You can associate it to the default image viewer for all types of images. If you find that it is not appearing in the Open With menu, then open /usr/share/applications/feh.desktop. Add %U to the Exec line.

  • When you open an image in your file explorer with Feh, it only displays that image. You cannot move to the previous and next image in that directory. If you want Feh to act like an image browser, use the script provided here. To associate this script with your file explorer, first add it as a main menu entry and then find it in the Open With dialog.

Shortcomings

Though Feh is super fast, there are a few problems which make it impossible for me to use it:

  • It does not auto-rotate images to match their EXIF orientation. Since most images are photos generated by cameras or smartphones, not having this feature is a deathblow. I cannot rotate my head while browsing a directory of photos.

  • Manually rotating the image modifies the image! You can rotate the image by 90 degrees using ``, but it actually writes back the rotated image to file! There is no option to just rotate for viewing! This combined with Feh not supporting EXIF rotation tags is a deathblow.

Tried with: Feh 2.9.3 and Ubuntu 14.04
