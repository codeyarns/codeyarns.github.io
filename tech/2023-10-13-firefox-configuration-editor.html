Firefox Configuration Editor

The Configuration Editor in Firefox is invoked by using about:config as the URL in the addressbar. It is used to change the settings of the browser well beyond what is available in the Settings dialog. Any preference that is changed away from its default value is saved by Firefox to a file named prefs.js in its Profiles directory. This is a file that is written by Firefox and should not be modified by the user. The user can provide Firefox his own settings in a user.js, also placed in the Profiles directory. The user.js preferences will override that in prefs.js or the default.

This post captures some of these settings that I have found to be useful since 2009:

app.update.interval

The interval in seconds at which Firefox checks for updates. Set it to 1296000 for example to check once every 15 days.

browser.backspace_action

Set the above option to 0 to use the Backspace key to get to the Previous page.

While this works by default on Windows, that is not the case on Ubuntu. An alternative to this is Alt + ‚Üê

browser.ctrltab.previews

Set to true to display a Windows task-manager-like previews of open tabs when Ctrl+Tab is used to switch or cycle through the tabs.

browser.link.open_newwindow.restriction

It is irritating when clicking a link pops open a new window, because it also takes away the focus from the current window.

Set the above config to 0 to stop this behaviour and open all popup windows in a new tab.

browser.search.context.loadInBackground

In recent versions of Firefox, if you select any text, right-click and choose Search Google (or any search engine), the new tab with these results is opened in the foreground.

This default behaviour can be changed to load the tab in the background by setting the above config to true.

browser.search.defaultenginename

If you type some words into the address bar instead of a URL, Google is used as the default search engine to search with those words as keywords. Change the above config to the name of the default search engine you want like. For example, changing Google to Bing.

browser.tabs.closeButtons

To remove the tab close button, set this config to 2.

Firefox displays the close button on each open tab. This is a wastage of the horizontal space in the tab bar. You can close a tab anyway by using the keyboard or mouse as described in this post.

browser.tabs.loadDivertedInBackground

Toggle this to true to open clicked links in the background.

dom.disable_window_move_resize

Some webpages resize or move the browser window. Set the above config to true to disable this.

extensions.update.interval

The interval in seconds at which Firefox checks for updates to addons. Set it to 1296000 for example to check once every 15 days.

layout.word_select.eat_space_to_next_word

If you double-click anywhere on a word of text in any application on Windows, it will highlight that word and any following whitespace characters upto the beginning of the next word. Supposedly, Mac behaves the same way too.

The reason for this behaviour seems to be to aid in copy-paste of a word inside a document. It also helps if one wants to delete a word, since the following whitespace is also nicely removed.

However, I find this feature irritating. Especially when I am editing text on Firefox and want to apply formatting on the highlighted word either in rich text mode or HTML mode.

Thankfully, to highlight only the word on a double-click and ignore the following whitespace set the above config to false.
