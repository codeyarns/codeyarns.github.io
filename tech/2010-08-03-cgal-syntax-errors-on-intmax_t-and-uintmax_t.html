CGAL: Syntax Errors on intmax_t and uintmax_t

📅 2010-Aug-03 ⬩ ✍️ Ashwin Nanjappa ⬩ 🏷️ boost, cgal, errors, mpfr ⬩ 📚 Archive

Trying to compile CGAL 3.6.1 with Boost 1.42 on Windows generated a lot of errors of this form:

CGAL-3.6.1 .h(280) : error C2061: syntax error : identifier ‘intmax_t’ CGAL-3.6.1 .h(283) : error C2061: syntax error : identifier ‘uintmax_t’

The errors are related to the (missing) declaration of intmax_t and uintmax_t. According to this thread on the MPFR mailing list, these errors are caused due to changes in Boost 1.42 and later. So, until MPFR is updated to handle these Boost versions, the solution is to use older versions of Boost. I used Boost 1.41 and CGAL compiled successfully! 😊
