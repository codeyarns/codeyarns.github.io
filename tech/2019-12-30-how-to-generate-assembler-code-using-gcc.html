How to generate assembler code using gcc

(First posted on: 2019-12-30T23:51:23.970296-08:00)

When we use GCC to compile C or C++ code, it directly generates the final object or executable file. For example, this command generates an a.out executable:

$ g++ foobar.cpp

I find it useful to generate assembler code to understand what the compiler is doing to my C++ code. To generate assembler code we essentially request GCC to stop before the assembly stage of compilation and dump what it has generated from the compiler backend.

To generate assembler code:

$ g++ -S foobar.cpp

This writes the assembler code to a foobar.s file. For x86 and x64 assembler code, the AT&T syntax is used by default.

To generate assembler code in Intel syntax:

$ g++ -S -masm=intel foobar.cpp

The assembler code generated by the above -S option does not have any annotations indicating which lines of the C++ code correspond to which assembler instructions.

To add extra commentary to the assembler code:

$ gcc -S -fverbose-asm foobar.cpp

This option adds two types of commentary. First, every statement of the C++ code appears in the assembler code. This makes it easy to identify which assembler code lines correspond to every C++ source line. Note that function definition lines from C++ do not appear though, but I can usually identify them by looking at their mangled label name. Second, every assembler code line that uses registers has a comment to its right indicating which variable from the C++ code is being read or written. I find this information takes off a huge burden when reading assembler code.

To get elaborate commentary in the assembler code:

$ gcc -g -Wa,-adhln foobar.cpp > foobar.s

The -adhln options are passed to the assembler since we prefixed it with -Wa. The -adhln options essentially ask the assembler to enable assembler code listings (-a), omit debugging directives (-d), include C++ source code (-h), include assembly (-l) and omit other stuff (-n).

The above option provides the most pleasing annotations for assembler code. Unlike the -fverbose-asm option, this one includes all the C++ source lines including function definition lines. However, this invocation does not have commentary beside each assembler line indicating which C++ variable is being held in registers in that line.

Thankfully, all these options can be combined to generate the most detailed annotated assembler code:

$ gcc -g -Wa,-adhln -fverbose-asm foobar.cpp > foobar.s

Tried with: GCC 7.4.0 and Ubuntu 18.04
