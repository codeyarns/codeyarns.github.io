VirtualFish

📅 2023-Oct-05 ⬩ ✍️ Ashwin Nanjappa ⬩ 🏷️ fish, python ⬩ 📚 Archive

VirtualFish provides a vf command that makes working with Python virtual environments a lot easier with the Fish shell.

  • Install: Install the Python package for the user:
$ python3 -m pip install --user virtualfish

After that, install into the Fish shell:

$ vf install
$ exec fish
  • Uninstall: First we uninstall from the Fish shell and after that we remove the Python package:
$ vf uninstall
$ exec fish
$ python3 -m pip install virtualfish
  • Create:

Create a new virtual environment that is added under ~/.virtualenvs/ by default:

$ vf new my_venv

I could not find an option to create a virtual environment in the current directory. This was important to me since I wanted to maintain a virtual environment in every Python project I had, all the environments with the same name. Since this was not possible, this became the reason why I stopped using VirtualFish.

  • List: List all the virtual environments that are created and managed by VirtualFish (other virtual environments are not listed):
$ vf ls
my_venv
  • Activate:
$ vf activate my_venv
  • Deactivate:
$ vf deactivate my_venv
  • Remove:

Delete a virtual environment that was created by VirtualFish:

$ vf rm my_venv

Tried with: VirtualFish 2.5.5, Python 3.10 and Ubuntu 22.04
