Integer representations

(First posted on: 2020-04-05T21:25:54.712722-07:00)

  • In C/C++, both signed and unsigned integer types are available. Other languages like Java have only signed integer types.
  • Unsigned integer in almost all languages and computers have a single possible implementation show in the above figure.
  • Signed integers have 3 known implementations: two's complement, ones' complement and sign magnitude. Two's complement is what is universally used in computers today. Ones' complement used to be used in computers until the 1960s, like the CDC-160 for example. Sign magnitude was also used in computers until the 1960s, and is only used in the IEEE 954 floating point format now.
  • My mental model of how bits are interpreted in these 3 implementations is depicted in the above figure using a 4-bit integer type as an example.
  • Two's complement is a bit unintuitive for humans. But it won the race because addition, subtraction and multiplication are the simplest to implement in it and are the same for both unsigned and signed integers. For example, addition of unsigned integers and signed integers (in two's complement form) is the same bit-wise process: add the bits from LSB to MSB and ignore the final carry bit. So simple!
  • Two's complement, by its nature, has an uneven range. For example, 8-bit signed integer has range -128 to +127. Notice how negating -128 will not give you +128 because that value cannot be represented in a two's complement 8-bit signed integer. Computers using two's complement choose to return -128 as the negated value of -128.
  • Ones' complement, by its nature, has an even range. For example, 8-bit signed integer has range -127 to +127. But it will have two zeroes: +0 (when all bits are zero) and -0 (when all bits are one).
  • Sign magnitude, by its nature, has an even range. For example, 8-bit signed integer has range -127 to +127. But it will have two zeroes. You can see this in the IEEE 754 floating point format, which has two zeroes.
  • C/C++ standard allows signed integer to be in any of these 3 implementations in a bid to support old computers of the 1960s. This is the reason why the minimum ranges of signed integers in C/C++ standard have an even range. For example, the minimum range of 8-bit signed integers in C/C++ is -127 to +127. But implementations are allowed to have signed integer types whose range is larger, which is what all computers do today: their 8-bit signed integers have range -128 to +127 because they use two's complement.
