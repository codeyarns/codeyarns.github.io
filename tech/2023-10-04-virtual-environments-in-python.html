Virtual Environments in Python

Python programs running on real systems have 2 problems with packages. First, there is always a mix of packages to manage and it is a mess: the OS Python packages (installed using apt for example) and the local Python packages (installed using pip). A second problem is the inability to keep separate versions of packages for each program. The solution that is built into Python for these problems is virtual environments.

  • Install: To get started install the venv package:
$ sudo apt install python3-venv
  • Create: Go the root of any Python project for which you want an unique virtual environment and create it with a name you prefer:
$ python3 -m venv my_venv

This creates a my_venv subdirectory in the current directory with all the files required to use the virtual environment. This is also the directory where packages will be installed for this project.

It is a good idea to ignore all the files inside the virtual environment directory in Git by adding this line to .gitignore: my_venv/.

  • Activate: Enable the virtual environment by activating it. This is done by sourcing a shell script that is present in the my_venv directory.
$ source my_venv/source/activate

$ source my_venv/source/activate.fish

You are now inside a fresh virtual environment where there is just Python and its standard packages and nothing else. There is none of the OS/apt packages or packages you had installed using pip outside.

  • Install Python packages: Inside the activated virtual environment, install just the Python packages you need for this particular project using any method that suits you:
$ python3 -m pip install some_package

$ python3 -m pip install -r requirements.txt
  • Run: Run the Python program as normal inside the activated virtual environment. It will use only the Python packages installed inside the virtual environment.
$ python3 my_program.py
  • Deactivate: Disable the virtual environment at the shell once you are done and want to move to a different directory to do something else:
$ deactivate
  • Delete: There does not seem to be a command to delete a virtual environment. I delete the directory itself:
$ rm -rf my_venv
