How to setup SSH keys for multiple Github accounts

Problem

A local Git repository can push and pull from a remote Github account using SSH authentication. This is typically configured by adding your SSH key to your Github account. If you have two or more Github accounts and try to use the same SSH public key for them, Github rejects the key with the error Key is already in use.

Solution

This problem can be solved by following these steps:

  • Generate a SSH key for each Github account as described here. For example, generate ~/.ssh/id_ed25519.github.com-joe1 and ~/.ssh/id_ed25519.github.com-joe2.

  • Add the respective .pub keys to each of the Github accounts.

  • Add host entries in ~/.ssh/config for each Github account separately and associate it with its SSH key. For example:

# Github account: joe1
Host github.com-joe1
    HostName github.com
    User git
    AddKeysToAgent yes
    IdentityFile ~/.ssh/id_ed25519.github.com-joe1

# Github account: joe2
Host github.com-second
    HostName github.com
    User git
    AddKeysToAgent yes
    IdentityFile ~/.ssh/id_ed25519.github.com-joe2
  • Clone repos using the new host names shown above. For example:
$ git clone git@github.com-joe1:joe1/some_repo.git

$ git clone git@github.com-joe2:joe2/some_other_repo.git
  • For existing repos, change the SSH path in the .git/config file.

Tried with: Git 2.17.1 and Ubuntu 18.04
