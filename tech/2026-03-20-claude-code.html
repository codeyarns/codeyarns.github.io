Claude Code

📅 2026-Mar-20 ⬩ ✍️ Ashwin Nanjappa ⬩ 🏷️ ai ⬩ 📚 Archive

Claude Code is the CLI agentic AI tool by Anthropic designed to aid in software development. At the shell, this tool is invoked as claude.

Basics

  • Invoke claude at the root of any project directory.

  • Ask it questions in natural language about the codebase in that directory to learn more about the project.

  • Implement code changes by asking it in natural language. It will show you the diff before writing the changes to the files.

  • Commit the code changes to Git using natural language.

  • Drop into shell by suspending Claude Code using ctrl z. This is useful to run shell commands. Jump back into the suspended Claude session using fg.

  • Clear the current conversation history with /clear.

  • Everytime Claude Code is invoked, it does not remember the previous session. To make it resume the most recent session in that directory, use claude -c.

Built-in commands

  • /btw: Ask a side question that does not add to the current context window.

  • /compact: Compress the context window by replacing with a summary.

  • /context: Display visualization of current model info, context window usage, memory used by CLAUDE.md, AGENTS.md and skills.

  • /cost: Show total cost in $ from token usage.

  • /fast: Check if fast mode is ON or OFF. Also use to toggle it.

  • /init: Initialize the project with CLAUDE.md if it does not exist already.

  • /insights: Generates a report of your Claude Code usage at ~/.claude/usage-data/report.html. Be prepared to be severely judged and reviewed.

Skills

A skill is a way for the user to extend the toolkit of Claude Code with additional and specific functionality.

A skill is specified in a SKILL.md file like this:

---
name: use-super-power
description: Use the super power of Unix tools to answer my query.
---

Here is a list of Unix tools you should invoke when answering the query:
- du
- df

The name is the name of the skill, written typically in lowercase, no spaces and separated by hyphens. For example use-super-power. User can explicitly invoke it as /use-super-power.

The description is a user-friendly description of the skill in English. The skill will be used by Claude Code automatically if user asks something that is similar to what is in the description.

This file can be located for use by any project as ~/.claude/skills/use-super-power/SKILL.md. Or it can be located inside a specific directory for use only by that project: /home/joe/my-gnu-kernel/.claude/skills/use-super-power/SKILL.md. Other files required by the skill can be placed alongside in its directory.

To invoke a skill explicitly, use its name: /use-super-power. This loads the skill and also runs its with any input argument provided.

If the skill is loaded based on user’s textual description, Claude Code shows that the skill is being loaded:

Skill(use-super-power)
  ⎿  Successfully loaded skill

The loaded skills at any point in time can be seen by typing /.

Currently there is no way to ask Claude to load skills located in a specific random directory.

Tried with: Claude Code 2.1.79
