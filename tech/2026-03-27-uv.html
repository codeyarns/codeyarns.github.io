uv

📅 2026-Mar-27 ⬩ ✍️ Ashwin Nanjappa ⬩ 🏷️ python ⬩ 📚 Archive

uv is a package and project manager for Python projects. It fully replaces two things: manually using virtual environments and managing packages. It is super fast, ironically because it is written in Rust!

Install

$ curl -LsSf https://astral.sh/uv/install.sh | sh

Usage

  • Initialize the directory of the Python project by invoking this inside the directory:
$ uv init

This creates a pyproject.toml with project metadata and .python-version.

  • Add a package that this project depends on:
$ uv add markdown
  • Remove a package from this project:
$ uv remove markdown
  • Run the program:
$ uv run foobar.py

This automatically creates a virtual environment with directory .venv (if it does not already exist) and uv.lock.

  • To commit the uv files in the project directory to Git:
$ git add pyproject.toml uv.lock
  • To migrate an older project (that uses explicit venv and pip) to uv:
$ /bin/rm -f old_venv_dir

$ uv init

$ uv add -r requirements.txt

$ /bin/rm requirements.txt
  • To remove uv from a project directory:
$ rm -rf pyproject.toml uv.lock .python-version .venv
  • List the Python versions and their locations:
$ uv python list
  • Update itself:
$ uv self update

Self contained scripts

For single file scripts, it is easier to put the package dependencies metadata in the script itself. uv can support this in a way that is compliant with PEP 723.

  • Use uv to add the packages to the script:
$ uv add --script foobar.py markdown

This adds the markdown package as a dependency at the top of the foobar.py script.

  • Make the script executable and running it is as simple as:
$ ./foobar.py

How is this working with no extra directories and files added in our project directory? uv manages all the metadata and packages for this script in a global cache, separately for each script.

  • To remove a package as dependency of the script:
$ uv remove --script foobar.py markdown
  • To view tree of package dependencies:
$ uv tree --script foobar.py

Cache

The cache is where uv maintains the package and Python information for scripts where we inline the metadata.

  • To see where this cache is located:
$ uv cache dir
  • To prune the cache:
$ uv cache prune
  • To empty the cache:
$ uv cache clean

Tried with: uv 0.11.2
